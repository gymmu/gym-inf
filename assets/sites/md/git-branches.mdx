## Versionsverzweigungen (`branches`)

**Git** operiert auf sogenannten `branch`es. Ein `branch` ist ein Zweig im
Versionsverlauf, was nichts anderes bedeutet als dass alle `commit`s auf diesem
Zweig angeordnet sind.

Man kann in **Git** mehr wie nur einen `branch` haben, dadurch können mehrere
Personen an einem Projekt arbeiten, oder man kann verschiedene Entwicklungen zur
gleichen Zeit erproben. Dies kann praktisch sein, wenn man verschiedene Designs
testen möchte.
