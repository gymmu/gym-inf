Das Internet besteht aus ganz vielen verschiedenen Teilen. Wir schauen uns eine
vereinfachte Darstellung davon an und versuchen die wichtigsten Teile zu
verstehen.

<div style={{ display: "flex", justifyContent: "center", margin: "2rem 0" }}>
  <img src="./svg/network-diagram.svg" alt="Network Diagram" width="100%" />
</div>

#### Webserver

Das sind Computer die irgendwo am Netzwerk angehängt sind, und über eine
öffentliche **IP-Adresse** verfügen. Ein Webserver nimmt Anfragen von einem
Client entgegen, und schickt meistens eine Webseite, oder einfach nur Daten
zurück.

#### DNS-Server

Das sind einige wenige spezielle Server die auf der ganzen Welt verteilt sind.
Ein DNS-Server speichert welche **IP-Adresse** zu welchem **Domain-Name** passt.
Wenn man also die **IP-Adresse** von `github.com` nicht auswendig weiss, kann
man das beim **DNS-Server** anfragen. Das passiert automatisch, und wird dann
auf allen Zwischengeräten gespeichert, so dass nicht jedes mal eine Anfrage an
den **DNS-Server** gemacht werden muss.

**DNS** steht übrigens für Domain-Name-System.

#### Router

Die **Router** sind das Herzstück vom Internet. Damit Daten durch das Internet
geschickt werden können, müssen mehrere **Router** miteinander verbunden sein.
Ein Router merkt sich dann immer mit welchen Geräten er verbunden ist. Wenn nun
einen Anfrage kommt, kann der Router diese gezielt an den nächsten Router
weiterleiten, oder wenn es eine ganz neue Verbindung ist, kann er einfach an
alle weiterleiten die er kennt.

#### IP-Adresse

Alle Geräte die im Internet sind, haben eine **IP-Adresse**. Die Geräte
untereinander sprechen sich über die **IP-Adresse** an, nicht über den
**Domain-Name**, so wie wir das machen würden.

Eine **IPv4-Adresse** besteht aus vier Zahlen, die durch Punkte getrennt sind.
Jede Zahl kann einen Wert von 0 bis 255 haben. Eine IPv4-Adresse besteht aus 4
Bytes (32 Bit). Beispiele für IPv4-Adressen sind:

- **Localhost:** 127.0.0.1 (wird verwendet, um auf den eigenen Computer zu
  verweisen)
- **Lokales Netzwerk:** 192.168.1.1 (häufig verwendete Adresse für Router in
  privaten Netzwerken)
- **DNS-Server:** 8.8.8.8 (ein öffentlicher DNS-Server von Google)

#### Modem / ISP

Ein **IPS** ist Ihr Internet-Service-Provider, also die Firma die Ihnen den
Anschluss and Internet anbietet. Bei den meisten wird das _Swisscom_, _Sunrise_
oder _Cablecom_ sein. Diese Firma stellt Ihnen ein **Modem** zur Verfügung,
welches über das Internet erreichbar ist (es hat eine öffentliche IP-Adresse).
Das **Modem** funktioniert dann wie ein **Router**. Es kennt alle Geräte im
lokalen Netzwerk, und kann Daten zu diesen schicken. Wenn Daten nach draussen
sollen, dann schickt es diese einfach zum nächsten **Router**. Der ist dann
meistens beim **ISP** selbst, und von dort werden die Daten dann weiter
geleitet.

Das heisst also, Ihr **ISP** weiss immer ganz genau welche Webseiten Sie
besuchen. Mit einem **VPN** kann man sich davor schützen, die Daten liegen dann
aber beim **VPN**.

#### Client

Das sind die Endgeräte, wie Laptops und Smartphones die an einem lokalen Netz
angehängt sind. Wenn wir zwischen 2 Geräten kommunizieren möchten, dann geht das
immer über ein weiteres Gerät (z.B. das **Modem**). Es ist schon auch möglich
eine wirklich direkte Verbindung aufzubauen, aber das ist oftmals zu mühsam. Nun
sollte auch klar sein, dass alle Verbindungen und Daten im Internet abgehört
werden können. Denn alle Geräte sind miteinander verbunden. Wenn man es schaft
sich bei einer Verbindung dazwischen zu schalten, dann kann man den ganzen
Verkehr auf dieser Verbindung abhören und auswerten. In diesem Fall spricht man
von einer **man-in-the-middle**-Attacke. Um sich davon zumindest teilweise zu
schützen, ist es wichtig das nur Verbindungen über **HTTPS** verwendet werden.

<h2 id="vpn">Anonym im Internet (VPN)</h2>

Sie haben sicher schon davon gehört das Sie mit einem **VPN** anonym und sicher
im Internet surfen können. Sie können auch ganz eifach den Standort wechseln
wenn Sie ein **VPN** verwenden. Schauen wir uns doch zusammen an, was es damit
auf sich hat.

Zuerst müssen Sie verstehen wie man einem Gerät im Internet einen geografischen
Standort zuweisen kann. Das ist ganz einfach, Ihr **ISP** hat das in einer
Datenbank registriert. Wie genau die Adresse tatsächlich ist, hängt vom **ISP**
ab. Das kann Ihre exakte Adresse sein, oder einfach nur der Ort in dem Sie
registriert sind. Das reicht aber schon aus, um gewisse Inhalte für Sie zu
blockieren. Das passiert ganz oft bei _Netflix_ oder ähnlichen anbietern, wenn
es wegen der Lizenz nicht möglich ist einen Inhalt in anderen Ländern zu
publizieren. Um das zu umgehen, verwenden viele ein **VPN**. Aber was passier
bei einem **VPN** denn genau.

Ein **VPN** ist ein Programm das bei Ihnen auf dem Computer läuft. Bevor irgend
ein Datenpacket verschickt wird, wird das Packet verschlüsselt, dann wird das
verschlüsselte Packet an den **VPN**-Anbieter geschickt. Dort wird es entpackt
und dann erneut an das eigentliche Ziel verschickt. Sie sieht es für den
Empfänger so aus, als ob die Daten vom **VPN**-Server direkt kommen, und nicht
von Ihnen. Auf dem Rückweg werden die Daten dann wieder zum **VPN** geschickt,
wieder verschlüsselt, und zurück an Sie geschickt.

Wenn jemand (Ihr **ISP** zum Beispiel) diese Verbindung nun abhört, sieht man
nur dass Sie mit dem **VPN** kommunizieren. Ihr **ISP** kann Sie also nicht mehr
ausspionieren. Früher (heute noch immer) wird diese Technik gerne verwendet um
sich in Firmennetze zu verbinden. Daher der Name **virtual private network**.

Ob Sie nun mit einem **VPN** tatsächlich anonym im Internet unterwegs sind,
können Sie sich sehr einfach ausdenken. Ihr **ISP** und alle anderen die
mithören würden, können Ihre Verbindungen nicht mehr lesen, oder das Ziel davon
sehen. Wenn es vom **VPN** weitergeht, kann das Ziel nicht sehen wer der
Absender war (ausser natürlich Sie sind mit einem Account eingeloggt). Es wirkt
also als seien Sie tatsächlich anonym, ausser natürlich für den
**VPN**-Anbieter. Da müssen Sie sich dann fragen wie sehr Sie dem Vertrauen
möchten. denn nimmt man das mit der _No Log Policy_ wörtlich, merken Sie ganz
schnell das es technisch garnicht möglich ist. Bis eine Antwort zurück zum
Absender geschickt wurde, muss ein **VPN** speichern woher die Anfrage gekommen
ist, und wo sie hingeht, sonst kann die Antwort nicht zugestellt werden. Danach
kann man aber alles löschen. Ob das tatsächlich gemacht wird, ist aber fraglich.
Denn Ihr **ISP** behält all Ihre Verbindungsdaten über mehrere Jahre. Technisch
ist es also sicher möglich solche grossen Datenmengen zu speichern.

Man kann also sagen dass Sie ein wenig anonymer im Internet unterwegs sind, wenn
Sie ein **VPN** verwenden. Die Problematik wer all Ihre Verbindungsdaten hat,
verschiebt sich dann einfach vom **ISP** zum **VPN**.

<Example id="nat" title="NAT (Network Address Translation)">

Es git ein sehr grosses Problem mit den **IP-Adressen**. Es hat nicht genügend
Adressen für alle Geräte, deshalb hat man sich einen Tick mit **NAT** einfallen
lassen.

In einem lokalen Netzwerk haben viele Geräte private IP-Adressen, die nur
innerhalb dieses Netzwerks gültig sind. Damit diese Geräte trotzdem mit dem
Internet kommunizieren können, wird **NAT** verwendet.

**NAT** steht für _Network Address Translation_. Es ist eine Technik, bei der
ein Router die privaten IP-Adressen der Geräte im lokalen Netzwerk in eine
öffentliche IP-Adresse übersetzt, die im Internet verwendet wird.

Wenn ein Gerät im lokalen Netzwerk eine Anfrage ins Internet schickt, ersetzt
der Router die private IP-Adresse durch seine eigene öffentliche IP-Adresse. Die
Antwort vom Internet wird dann vom Router wieder an das richtige Gerät im
lokalen Netzwerk weitergeleitet.

Dadurch können viele Geräte in einem lokalen Netzwerk dieselbe öffentliche
IP-Adresse teilen, was die Anzahl der benötigten öffentlichen IP-Adressen
reduziert.

Ein weiterer Vorteil von **NAT** ist, dass es eine Art Schutz bietet, da Geräte
im lokalen Netzwerk von aussen nicht direkt erreichbar sind.

<h2 id="firewalls">Firewalls</h2>

Firewalls sind Sicherheitsmechanismen, die den Datenverkehr zwischen
verschiedenen Netzwerken kontrollieren. Sie entscheiden, welche Verbindungen
erlaubt sind und welche blockiert werden.

Eine Firewall kann entweder als Software auf einem Gerät oder als eigenständiges
Gerät im Netzwerk eingesetzt werden.

Firewalls schützen Netzwerke vor unerwünschtem Zugriff von aussen und helfen
dabei, schädliche Datenpakete zu erkennen und zu blockieren.

Sie sind ein wichtiger Bestandteil der Netzwerksicherheit und tragen dazu bei,
dass nur vertrauenswürdige Verbindungen zugelassen werden.

Wenn Sie einen Server bei sich zu Hause laufen lassen möchten, damit andere auf
Ihren Server zugreifen können, müssen Sie die nötigen Ports in der Firewall
freischalten. Das machen Sie auf dem **Modem** und der Vorgang nennt sich dort
oftmals _Port-Forwarding_. Sie sagen dem Gerät also dass Verbindungen auf einem
bestimmten Port, zum Beispiel 5173 an eine lokale **IP-Adresse** geschickt
werden soll. So verwenden Sie dann **NAT** um Ihren Server für das ganze
Internet bereitzustellen.
