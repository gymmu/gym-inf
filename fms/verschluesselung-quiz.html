Verschlüsselung und Schlüsselaustausch Quiz

Testen Sie Ihr Wissen über Verschlüsselung, das Schlüsselaustausch-Problem und HTTPS!

Dieses Quiz basiert auf den Inhalten des Verschlüsselungs-Kapitels. Falls Sie Schwierigkeiten mit den Fragen haben, schauen Sie sich das Kapitel noch einmal an.

Was ist das Hauptproblem beim Schlüsselaustausch?

  1. Schlüssel sind zu lang
  2. Wie tauscht man einen geheimen Schlüssel sicher über eine unsichere Verbindung aus?
  3. Verschlüsselung ist zu langsam
  4. Es gibt zu viele verschiedene Verschlüsselungsverfahren

Bei der symmetrischen Verschlüsselung wird der gleiche Schlüssel zum Ver- und Entschlüsseln verwendet.

  1. Wahr
  2. Falsch

Was ist der Hauptvorteil der asymmetrischen Verschlüsselung?

  1. Sie ist schneller als symmetrische Verschlüsselung
  2. Sie löst das Schlüsselaustausch-Problem
  3. Sie braucht keine Schlüssel
  4. Sie ist einfacher zu implementieren

Im Vorhängeschloss-Prinzip entspricht der öffentliche Schlüssel...

  1. dem Schlüssel zum Öffnen des Schlosses
  2. dem offenen Vorhängeschloss
  3. der verschlossenen Kiste
  4. der geheimen Nachricht

Der private Schlüssel wird beim Verbindungsaufbau über das Internet geschickt.

  1. Wahr
  2. Falsch

Welche Verschlüsselungsart verwendet HTTPS für den eigentlichen Datenaustausch?

  1. Nur asymmetrische Verschlüsselung
  2. Nur symmetrische Verschlüsselung
  3. Keine Verschlüsselung
  4. Abwechselnd beide

Warum kombiniert HTTPS asymmetrische und symmetrische Verschlüsselung?

  1. Weil es gesetzlich vorgeschrieben ist
  2. Asymmetrisch für sicheren Schlüsselaustausch, symmetrisch für schnelle Datenübertragung
  3. Um mehr Sicherheit durch Redundanz zu haben
  4. Weil Browser nur diese Kombination unterstützen

Symmetrische Verschlüsselung ist schneller als asymmetrische Verschlüsselung.

  1. Wahr
  2. Falsch

Was garantiert ein digitales Zertifikat?

  1. Dass die Webseite keine Viren hat
  2. Dass der öffentliche Schlüssel wirklich zur Webseite gehört
  3. Dass die Webseite kostenlos ist
  4. Dass die Verschlüsselung besonders stark ist

Certificate Authorities (CAs) stellen digitale Zertifikate aus.

  1. Wahr
  2. Falsch

Was passiert in Phase 1 des HTTPS-Handshakes?

  1. Die eigentlichen Daten werden übertragen
  2. Ein symmetrischer Schlüssel wird sicher ausgetauscht
  3. Das Zertifikat wird erstellt
  4. Die Webseite wird geladen

Was ist das Man-in-the-Middle Problem?

  1. Der Server ist zu langsam
  2. Ein Hacker stellt sich zwischen Client und Server und fängt Daten ab
  3. Die Verschlüsselung funktioniert nicht richtig
  4. Das Zertifikat ist abgelaufen

Bei asymmetrischer Verschlüsselung kann jeder, der den öffentlichen Schlüssel hat, damit verschlüsselte Nachrichten entschlüsseln.

  1. Wahr
  2. Falsch

Wofür wird der öffentliche Schlüssel verwendet?

  1. Zum Entschlüsseln von Nachrichten
  2. Zum Verschlüsseln von Nachrichten
  3. Zum Signieren von Zertifikaten
  4. Zum Löschen von Nachrichten

Was prüft Ihr Browser, wenn er ein Zertifikat erhält?

  1. Nur ob das Zertifikat existiert
  2. Ob es von einer vertrauenswürdigen CA ausgestellt wurde, noch gültig ist und zum Domainnamen passt
  3. Nur die Gültigkeitsdauer
  4. Ob die Webseite schön aussieht

  • ✅ Die Rolle von digitalen Zertifikaten verstehen
