Quiz: Internet und Netzwerke

Testen Sie Ihr Wissen über das Internet, das TCP/IP-Modell, DNS-Auflösung, TCP vs. UDP und mehr!

Dieses Quiz basiert auf den Inhalten des Internet-Kapitels. Falls Sie Schwierigkeiten mit den Fragen haben, schauen Sie sich das Kapitel noch einmal an.

Welche Aussage beschreibt den Unterschied zwischen TCP und UDP am besten?

  1. TCP garantiert die Zustellung und Reihenfolge der Pakete, UDP nicht.
  2. UDP ist langsamer als TCP, dafür sicherer.
  3. TCP wird für Streaming verwendet, UDP für E-Mails.
  4. UDP verwendet einen 3-Way-Handshake, TCP nicht.

In welcher Schicht des TCP/IP-Modells arbeitet das DNS-Protokoll?

  1. Anwendungsschicht
  2. Transportschicht
  3. Internetschicht
  4. Netzzugangsschicht

Was ist ein Root-Nameserver in der DNS-Hierarchie?

  1. Der oberste DNS-Server, der weiss, welche Nameserver für die TLDs (z.B. .com, .ch) zuständig sind.
  2. Der DNS-Server bei Ihrem ISP, der Anfragen für Sie ausführt.
  3. Der Nameserver, der die IP-Adresse einer konkreten Domain kennt.
  4. Ein DNS-Server, der nur im lokalen Netzwerk funktioniert.

HTTPS schützt vollständig davor, dass Ihr ISP sehen kann, welche Webseiten Sie besuchen.

  1. Wahr
  2. Falsch – der ISP sieht weiterhin die Domains (aber nicht den Inhalt).

Warum wird UDP für DNS-Anfragen verwendet?

  1. Weil DNS-Anfragen klein und schnell sind, und ein verlorenes Paket einfach erneut gesendet werden kann.
  2. Weil UDP verschlüsselter ist als TCP.
  3. Weil TCP von DNS-Servern nicht unterstützt wird.
  4. Weil UDP keine Ports verwendet und deshalb sicherer ist.

Was passiert beim TCP 3-Way-Handshake?

  1. Client sendet SYN → Server antwortet SYN-ACK → Client bestätigt ACK.
  2. Client sendet ACK → Server antwortet SYN → Client bestätigt FIN.
  3. Server sendet SYN → Client antwortet ACK → Server bestätigt FIN.
  4. Client und Server tauschen gleichzeitig SYN-Pakete aus.

Wie viele Hosts kann ein /26-Subnetz (255.255.255.192) enthalten?

  1. 62 Hosts
  2. 64 Hosts
  3. 126 Hosts
  4. 254 Hosts

IPv6-Adressen sind 128 Bit lang und lösen das IPv4-Adressknappheitsproblem.

  1. Wahr
  2. Falsch

Was ist DNS over HTTPS (DoH)?

  1. Ein Verfahren, das DNS-Anfragen über verschlüsselte HTTPS-Verbindungen sendet, sodass der ISP sie nicht mitlesen kann.
  2. Ein Protokoll, das Webseiten-Domains in HTTPS umwandelt.
  3. Ein DNS-Server, der nur HTTPS-Webseiten auflöst.
  4. Eine Erweiterung, die HTTPS-Zertifikate im DNS speichert.

Was zeigt der Befehl 'traceroute' an?

  1. Die Router (Hops), über die ein Paket vom eigenen Gerät bis zum Ziel läuft.
  2. Die Geschwindigkeit der eigenen Internetverbindung.
  3. Alle aktuell geöffneten Netzwerkverbindungen des eigenen Geräts.
  4. Die DNS-Auflösungszeit für eine Domain.

Welche Aussage über Datenpakete im Internet ist korrekt?

  1. Pakete einer Übertragung können verschiedene Wege nehmen und am Ziel wieder zusammengesetzt werden.
  2. Alle Pakete einer Verbindung nehmen immer den gleichen Weg.
  3. Pakete werden als ganzes Stück gesendet, nicht aufgeteilt.
  4. Nur TCP-Pakete können verschiedene Wege nehmen, UDP-Pakete nicht.

Was bedeutet 'BGP' im Kontext des Internets?

  1. Border Gateway Protocol — das Protokoll, mit dem grosse Netzwerke (Autonomous Systems) Routing-Informationen austauschen.
  2. Browser Gateway Protocol — ein Protokoll für sichere Browser-Kommunikation.
  3. Basic Gateway Protocol — ein vereinfachtes Routing-Protokoll für Heimnetzwerke.
  4. Backup Gateway Protocol — ein Sicherheitsprotokoll für Redundanz.

Zusammenfassung der Lernziele

  • Das TCP/IP-Schichtenmodell und die Aufgaben der vier Schichten kennen
  • Den Unterschied zwischen TCP (zuverlässig) und UDP (schnell) erklären können
  • Den vollständigen Ablauf einer DNS-Auflösung beschreiben können
  • Verstehen, was HTTPS schützt und was nicht
  • Die Bedeutung von NAT und IPv6-Adressierung kennen
