Quiz: Passwörter und Authentifizierung

Testen Sie Ihr Wissen über Hashing, Salting, Angriffsmethoden, Zwei-Faktor-Authentifizierung und FIDO2/Passkeys!

Dieses Quiz basiert auf den Inhalten des Passwörter-Kapitels. Falls Sie Schwierigkeiten mit den Fragen haben, schauen Sie sich das Kapitel noch einmal an.

Warum sind spezielle Passwort-Hash-Algorithmen wie bcrypt oder Argon2 besser als SHA-256 für das Speichern von Passwörtern?

  1. Weil sie absichtlich langsam sind, was Brute-Force-Angriffe massiv erschwert.
  2. Weil sie sicherere Hashes erzeugen, die nicht geknackt werden können.
  3. Weil sie automatisch salzen.
  4. Weil SHA-256 nicht für Passwörter zugelassen ist.

Was ist der Zweck von 'Salting' beim Speichern von Passwörtern?

  1. Ein einzigartiger zufälliger Wert wird ans Passwort angehängt, sodass gleiche Passwörter unterschiedliche Hashes erzeugen und Rainbow Tables unwirksam werden.
  2. Das Passwort wird vor dem Hashing verschlüsselt.
  3. Das Passwort wird verlängert, damit es schwerer zu erraten ist.
  4. Der Hash wird mit einem geheimen Server-Schlüssel kombiniert.

Was ist 'Credential Stuffing'?

  1. Gestohlene Benutzername-Passwort-Kombinationen aus einem Einbruch werden automatisiert bei anderen Diensten ausprobiert.
  2. Ein Angriff, der viele verschiedene Passwörter auf ein einzelnes Konto ausprobiert.
  3. Das Verstecken von Malware in scheinbar harmlosen Dateien.
  4. Ein Angriff auf die Datenbankstruktur eines Webservers.

Wie funktioniert TOTP (Time-based One-Time Password)?

  1. Server und App berechnen unabhängig denselben Code aus einem geteilten Secret Key und der aktuellen Zeit (30-Sekunden-Fenster).
  2. Der Server sendet einen Code per SMS, den der Nutzer eingeben muss.
  3. Ein zufälliger Code wird beim Login vom Server generiert und per E-Mail geschickt.
  4. Der Code wird aus dem Passwort des Nutzers berechnet.

SMS-basierte Zwei-Faktor-Authentifizierung gilt heute als sicher und wird von Sicherheitsexperten empfohlen.

  1. Wahr
  2. Falsch – SMS-2FA ist anfällig für SS7-Angriffe und SIM-Swapping.

Wie verhindert FIDO2/Passkeys Phishing-Angriffe?

  1. Die Domain ist in die Challenge eingebettet — eine Signatur für eine gefälschte Domain ist für die echte Domain wertlos.
  2. Passkeys blockieren alle Verbindungen zu unbekannten Webseiten.
  3. Passkeys verschlüsseln alle Kommunikation stark genug, dass Phishing nicht möglich ist.
  4. Passkeys erfordern eine zweite Bestätigung per E-Mail.

Wo wird der private Schlüssel eines Passkeys gespeichert?

  1. Im sicheren Element des Geräts (z.B. Secure Enclave, TPM-Chip) und verlässt das Gerät niemals.
  2. Beim Webserver als Teil des Kontos.
  3. In der Browser-History des Nutzers.
  4. Beim FIDO-Alliance-Server.

Was bedeutet 'Zero-Knowledge' bei einem Passwortmanager?

  1. Der Anbieter kann die gespeicherten Passwörter nicht lesen — selbst wenn sein Server gehackt wird, sind die Daten ohne das Master-Passwort wertlos.
  2. Der Passwortmanager speichert nichts und generiert Passwörter immer neu.
  3. Der Nutzer muss kein Passwort wissen, weil biometrische Daten verwendet werden.
  4. Das Passwort wird in der Cloud mit null Verschlüsselung gespeichert.

Bei einer Zero-Knowledge-Architektur eines Passwortmanagers ist ein vergessenes Master-Passwort in der Regel nicht wiederherstellbar.

  1. Wahr – ohne Master-Passwort kann der Vault nicht entschlüsselt werden.
  2. Falsch – der Anbieter kann immer helfen, das Passwort zurückzusetzen.

Welche Authentifizierungskategorie ist 'Fingerabdruck' oder 'Gesichtserkennung'?

  1. Inherence ('Etwas, das Sie sind')
  2. Knowledge ('Etwas, das Sie wissen')
  3. Possession ('Etwas, das Sie haben')
  4. Location ('Wo Sie sich befinden')

Welche Funktion hat PBKDF2 in einem Passwortmanager?

  1. Es leitet aus dem Master-Passwort und einem Salt einen starken Verschlüsselungsschlüssel ab und ist absichtlich langsam.
  2. Es verschlüsselt die gespeicherten Passwörter direkt.
  3. Es überprüft die Stärke eines eingegebenen Passworts.
  4. Es synchronisiert Passwörter zwischen Geräten.

Was ist der Hauptvorteil von Passkeys gegenüber Passwort + TOTP?

  1. Passkeys sind phishing-resistent durch Domain-Binding, während TOTP-Codes in Echtzeit weitergegeben werden können.
  2. Passkeys sind einfacher zu merken als ein langes Passwort.
  3. Passkeys benötigen keine Internetverbindung.
  4. Passkeys sind kostenlos, während TOTP-Apps kostenpflichtig sind.

Zusammenfassung der Lernziele

  • Hashing und Salting als sichere Passwort-Speicherung erklären
  • Die wichtigsten Angriffsmethoden und ihre Gegenmassnahmen kennen
  • TOTP und SMS-2FA unterscheiden und einschätzen können
  • Die Funktionsweise von FIDO2/Passkeys verstehen
  • Die technische Grundlage von Passwortmanagern kennen
